1. Why a commercial license exists
----------------------------------

PhoneBridge is open-sourced under the AGPL-3.0. The AGPL is a strong
copyleft license: if you distribute the software or run a modified
version on a network server, you must publish the complete
corresponding source code of your modifications under the same AGPL
terms. For many organizations that is not a viable option — for
example:

- You want to ship PhoneBridge (or a derivative) inside a proprietary
  device, appliance, or OEM firmware and not publish the device’s
  internal fork.
- You operate PhoneBridge as part of a hosted service and you are
  unwilling — or contractually unable — to publish your
  service-specific modifications.
- You need patent peace, indemnification, support SLAs, or a defined
  warranty that the AGPL explicitly disclaims.

For these cases the copyright holder can grant a separate written
commercial license that replaces the AGPL obligations with negotiated
terms.


2. What a commercial license typically grants
--------------------------------------------

A signed commercial license may, depending on what is negotiated:

- Permit distribution of PhoneBridge and/or derivative works in
  object-code form (e.g. embedded in proprietary firmware) without
  the AGPL’s source-disclosure requirement for the surrounding
  proprietary code.
- Permit network deployment of a modified PhoneBridge without
  publishing the modifications.
- Grant a covenant not to assert the licensor’s patent claims
  against the licensee for the licensed scope.
- Provide limited warranties, indemnification, and support commitments
  that the AGPL disclaims.
- Set a defined term, field of use, geography, and seat / device /
  deployment volume, with a corresponding license fee.

These are typical building blocks. The actual scope, exclusions,
term, and pricing are defined per agreement.


3. What a commercial license does NOT grant
-------------------------------------------

Unless expressly stated in the signed agreement, a commercial
license does NOT:

- Transfer ownership of the PhoneBridge source code or the
  PhoneBridge trademark to the licensee.
- Permit re-licensing of the unmodified PhoneBridge source code
  itself under a different license — the AGPL-3.0 still governs any
  copy of the source you obtained from this repository and choose to
  redistribute as open source.
- Cover third-party components used by PhoneBridge. Each such
  component remains governed by its own license (see `Cargo.lock` and
  the `THIRD_PARTY_NOTICES` file if present).
- Override the AGPL terms for anyone other than the named licensee.


4. How to obtain a commercial license
-------------------------------------

To start a conversation about a commercial license for PhoneBridge:

1. Open a GitHub issue on this repository with the label
   `commercial-license` and a short description of the intended use
   (device type, deployment size, fields of use, expected term).
2. The copyright holder will follow up with a non-binding discussion,
   a draft term sheet, and a quote.
3. A signed commercial license agreement is required before any
   rights under this addendum are effective.

Until a signed agreement is in place, the AGPL-3.0 (see `LICENSE`)
governs all use of this software.
